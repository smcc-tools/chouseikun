# 調整くん イベントハブ化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 日程調整からスケジュール専用にし、精算/割り勘/座席/お店検索をマイイベントから起動（親子リンク）できるハブにする。単体利用は温存。

**Architecture:** 単一 `index.html`（インラインJS, ESM Firebase v11, Firestore/Auth/FCM）。既存の別イベント方式（`events/{id}`, `type`別）を維持し、settle/walica/seating に任意フィールド `parentEventId` を追加。schedule イベント詳細（`viewCard`）にハブUIを足し、既存の作成フローを参加者投入＋`parentEventId`付きで起動する。

**Tech Stack:** Vanilla JS (ESM), Firebase v11 (firestore/auth/messaging) compat無, GitHub Pages, Cloud Functions(gen2, Node).

## Global Constraints

- 実装は `test/index.html`（必要なら `test/sw.js`）に対して行う。ルートの `index.html` は直接編集しない（[[test-first-deploy-workflow]]）。本番反映はユーザー承認後に test→root を promote。

- テストランナーは無い。各タスクの検証は「ローカル配信 `python3 -m http.server 8000 --directory 日程調整アプリ` → プレビューで DOM/画面確認」または「Firestore データ確認（Admin SDKスクリプト or コンソール）」で行う。

```js
// イベントハブから各ツールを起動（参加者を投入し、parentEventId をセット）
function launchFromEvent(tool, eventId, participants) {
  const names = (participants || []).slice();
  if (tool === 'settle')  { goToSettleSetup();  settleDraftMembers  = names.slice(); renderSettleDraft();  hubParentEventId = eventId; }
  else if (tool === 'walica')  { goToWalicaSetup();  walicaDraftMembers  = names.slice(); renderWalicaDraft();  hubParentEventId = eventId; }
  else if (tool === 'seating') { goToSeatingSetup(); seatingDraftMembers = names.slice(); renderSeatingDraft(); hubParentEventId = eventId; }
  else if (tool === 'gourmet') { goToGourmet(); }  // 保存なし・parentEventIdなし
}
```

- [ ] **Step 2: renderEventHub を実装**

```js
function renderEventHub(event, allMyEvents) {
  const hub = document.getElementById('eventHub');
  const isHost = currentUser && Array.isArray(event.ownerUids) && event.ownerUids.includes(currentUser.uid);
  if (!hub || event.type !== 'schedule' || !isHost) { if (hub) hub.style.display = 'none'; return; }
  hub.style.display = 'block';
  const parts = Object.keys(event.participants || {});  // ※既存の参加者データ形式に合わせる
  document.getElementById('hubSettle').onclick  = () => launchFromEvent('settle',  event.id, parts);
  document.getElementById('hubWalica').onclick  = () => launchFromEvent('walica',  event.id, parts);
  document.getElementById('hubSeating').onclick = () => launchFromEvent('seating', event.id, parts);
  document.getElementById('hubGourmet').onclick = () => launchFromEvent('gourmet', event.id, parts);
  const linked = (allMyEvents || []).filter(e => e.parentEventId === event.id);
  const label = { settle:'精算', walica:'割り勘', seating:'座席決め' };
  document.getElementById('hubLinked').innerHTML = linked.length
    ? linked.map(e => `<a class="hub-link" href="?event=${e.id}">${(label[e.type]||'')}：${escapeHtml(e.name || '(無題)')}</a>`).join('')
    : '<p class="empty-msg" style="font-size:.82rem;">まだありません。上のボタンから作成できます。</p>';
}
```

**Interfaces:**
- Consumes: `userEventsRef`, `getDocs`, `setDoc`, `updateDoc`, `currentUser`。
- Produces: `async function migrateEmbeddedSettles()` — 自分の schedule イベントで `settle` を持ち `settleMigrated` が無いものを、新 settle レコード（`type:'settle'`, `parentEventId`＝そのschedule, `ownerUids`同じ）として作成し、元 schedule に `settleMigrated:true` を付与。冪等。

- [ ] **Step 1: migrateEmbeddedSettles を実装**

```js
async function migrateEmbeddedSettles() {
  if (!currentUser) return;
  let snap; try { snap = await getDocs(userEventsRef(currentUser.uid)); } catch { return; }
  for (const d of snap.docs) {
    const ev = d.data();
    if (ev.type === 'schedule' && ev.settle && !ev.settleMigrated) {
      const newId = crypto.randomUUID();
      await setDoc(doc(db, 'users', currentUser.uid, 'events', newId), {
        type: 'settle', name: (ev.name ? ev.name + 'の精算' : '精算'),
        ownerUids: ev.ownerUids || [currentUser.uid],
        parentEventId: d.id, settle: ev.settle,
        participants: ev.participants || {}, createdAt: Date.now(),
      });
      await updateDoc(d.ref, { settleMigrated: true });
    }
  }
}
```

（`users/{uid}/events/{id}` のパスは既存 `userEventsRef` の構造に合わせる。settle レコードのフィールドは既存の type:settle 作成時の形に合わせる。）
